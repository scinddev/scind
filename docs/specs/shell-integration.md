**Version**: 1.0.0

## Overview

Contrail provides shell integration that enables context-aware Docker Compose interaction through the `contrail-compose` shell function. This function automatically injects the correct project name and compose files based on the current directory, providing a seamless development experience.

## Behavior

### Two Interfaces

Contrail provides two complementary interfaces:

1. **`contrail` CLI**: The primary command for managing workspaces, applications, ports, and configuration. Uses standard completion generated by Cobra.

2. **`contrail-compose` shell function**: A context-aware passthrough to `docker compose` that automatically injects the correct project name and compose files based on the current directory or explicit flags. Uses delegated completion from Docker's own completion scripts.

### Why This Separation Exists

- Docker Compose has ~20 subcommands with hundreds of flags and options
- Reimplementing completion for all of them would be significant work with no added value
- Users already know `docker compose` — we just make it workspace-aware
- Shell function + completion delegation achieves full docker compose completion "for free"

## Data Schema

### The `contrail compose-prefix` Command

This command is the bridge between the shell function and Contrail's context resolution. It outputs a docker compose command prefix that includes all necessary flags.

#### Behavior

```bash
$ cd ~/workspaces/dev/app-one
$ contrail compose-prefix
docker compose -p dev-app-one -f '/home/user/workspaces/dev/app-one/docker-compose.yaml' -f '/home/user/workspaces/dev/.generated/app-one.override.yaml'

$ cd ~/workspaces/dev
$ contrail compose-prefix -a app-two
docker compose -p dev-app-two -f '/home/user/workspaces/dev/app-two/docker-compose.yaml' -f '/home/user/workspaces/dev/app-two/docker-compose.worker.yaml' -f '/home/user/workspaces/dev/.generated/app-two.override.yaml'
```

#### Flags

| Flag | Short | Description |
|------|-------|-------------|
| `--workspace` | `-w` | Specify workspace (overrides context detection) |
| `--app` | `-a` | Specify application (overrides context detection) |

> **Note**: There is no `--flavor` flag. Flavor changes require regeneration and can impact running applications, so flavor must be set explicitly via `contrail flavor set` before using `contrail-compose`.

#### Output Format

- All paths are single-quoted to handle spaces and special characters
- Compose files are included in the correct order (base files first, then generated override)
- Project name follows the `{workspace}-{app}` convention
- Output is a single line suitable for `eval`

#### Exit Codes

| Code | Meaning |
|------|---------|
| 0 | Success - prefix output to stdout |
| 1 | General error (configuration issues, file not found, etc.) |
| 5 | Context detection failed (no workspace/app found) |

The shell function `contrail-compose` checks for empty output to detect failures and re-runs the command to display the error message. When `compose-prefix` succeeds, `contrail-compose` returns the exit code from the underlying `docker compose` command.

## Installation

```bash
# Bash
contrail init-shell bash >> ~/.bashrc

# Zsh
contrail init-shell zsh >> ~/.zshrc

# Fish
contrail init-shell fish >> ~/.config/fish/conf.d/contrail.fish
```

## Examples

### Basic Usage

```bash
# From within an application directory (context detected)
$ cd ~/workspaces/dev/app-one
$ contrail-compose exec php bash
$ contrail-compose logs -f
$ contrail-compose up -d

# From workspace root with explicit app
$ cd ~/workspaces/dev
$ contrail-compose -a app-two exec php bash
$ contrail-compose -a app-one logs -f php

# From anywhere with explicit workspace and app
$ contrail-compose -w dev -a app-one ps
```

### Tab Completion Examples

```bash
# Complete subcommands
$ contrail-compose [TAB]
build   config  create  down    events  exec    images  kill    logs ...

# Complete service names (from the resolved compose file)
$ contrail-compose exec [TAB]
php     nginx   mysql   redis

# Complete docker compose flags
$ contrail-compose exec -[TAB]
-d          --detach        -e          --env
--index     --no-tty        --privileged ...

# Complete with explicit app (contrail flags first)
$ contrail-compose -a app-[TAB]
app-one     app-two     app-three

$ contrail-compose -a app-two exec [TAB]
php     nginx   postgres    worker
```

## Edge Cases

### Context Resolution Failure

**Scenario**: Running `contrail-compose` outside any workspace.

**Behavior**:
```bash
$ cd ~
$ contrail compose-prefix
Error: No application context detected.

Either:
  1. Run from within an application directory
  2. Specify explicitly: contrail-compose -a APP [command]

Available workspaces: contrail workspace list
$ echo $?
5
```

## Known Limitations

### All Shells

1. **Docker completion must be installed**: If docker's shell completion isn't available, the delegation falls back to basic subcommand completion only.

2. **First invocation latency**: The first tab completion calls `contrail compose-prefix`, which involves context detection. Subsequent completions in the same directory use shell caching.

3. **Path spaces**: Paths with spaces are supported but require careful quoting in `contrail compose-prefix` output.

### Bash-Specific

1. **Requires bash-completion package**: The `_init_completion` helper function comes from bash-completion. Without it, completion may not work.

2. **COMP_LINE reconstruction**: Some docker completions use `COMP_LINE` for substring matching. Our reconstruction from words may lose original spacing.

### Zsh-Specific

1. **1-indexed arrays**: The completion function must account for zsh's 1-indexed arrays when calculating positions.

2. **Completion function detection**: Docker's completion might be `_docker`, `_docker-compose`, or loaded lazily. The script tries multiple approaches.

### Fish-Specific

1. **`complete -C` overhead**: Fish's completion delegation spawns a subshell, adding latency compared to bash/zsh.

2. **Completion caching**: Fish aggressively caches completions. If compose files change, completions may be stale until the shell restarts.

## Error Handling

| Error Condition | Exit Code | Message | Recovery |
|-----------------|-----------|---------|----------|
| No workspace found | 5 | `No workspace found in current directory or parents` | Navigate to workspace or use explicit flags |
| No app context | 5 | `No application context detected` | Navigate to app directory or use `-a` flag |
| Docker not available | 4 | `Docker is not installed or not running` | Start Docker |

## Future Considerations

### Potential Enhancements

1. **Completion caching**: Cache the compose-prefix output for a short duration to reduce latency on repeated completions.

2. **Service name completion from YAML**: Instead of delegating to docker for service names, parse the compose file directly. This would work even when containers aren't running.

3. **Flavor-aware completion**: Show available flavors when completing after `-f`/`--flavor` flag.

4. **Custom shortcuts**: Allow users to define additional shortcuts in Contrail config:
   ```yaml
   # ~/.config/contrail/config.yaml
   shortcuts:
     artisan: "exec php php artisan"
     composer: "exec php composer"
   ```

   Enabling: `contrail-compose artisan migrate` → `docker compose ... exec php php artisan migrate`

### Alternative Approaches Considered

1. **Integrated `contrail compose` subcommand**: Would require reimplementing all docker compose argument parsing and completion. Rejected due to complexity and maintenance burden.

2. **Alias-only approach** (`alias dc='docker compose -p ... -f ...'`): Doesn't support dynamic context detection. Rejected.

3. **Separate commands per docker compose subcommand** (`contrail-exec`, `contrail-logs`, etc.): Fragments the interface unnecessarily. Rejected.
